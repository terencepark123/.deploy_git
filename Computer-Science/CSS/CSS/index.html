CSS

1.CSS란?

HTML이 원래 정보 + 디자인의 기능을 하고 있었습니다. 하지만 두 기능을 분리해야 하는 니즈가 있었고, 이를 위해 기존의 HTML은 정보의 기능을 담당하고, CSS는 디자인의 기능을 담당하도록 분리되었습니다. 따라서 CSS는 HTML이 표현하는 정보를 꾸며주는 역할을 하게 됩니다.

CSS와 HTML이 만나는 법

  1. 인라인 방법 (HTML의 문법) - 태그의 속성에 직접 작성하여 직관적입니다.

  2. <head> 태그 내에 한꺼번에 정의하는 방법 - style 태그 안에 css를 작성합니다.

2.기본적인 문법

선택자

선택자와 선언

어떤 태그를 디자인하기 위해서는 디자인하려는 태그를

  1. 선택하고 (selector)
  2. 선택한 대상에게 효과를 줍니다. (declaration)

선택자의 타입들

  • 태그 선택자

    특정 태그에 공통적인 디자인을 적용합니다.

  • 아이디 선택자

    특정 요소에 디자인을 적용할 때 사용합니다. 주민등록번호처럼 고유해야 합니다.

  • 클래스 선택자

    여러 요소들에 공통적으로 적용할 수 있습니다.

부모 자식 선택자

CSS가 적용될 선택자를 더욱 세밀하게 선택할 수 있습니다.

  • 조상 자손 선택자

    밑에 있는 태그 모두를 선택합니다.

  • 부모 자손 선택자

    바로 밑의 태그를 선택합니다.

  • 동시 선택자

    두 태그를 동시에 선택합니다.

가상 선택자

클래스 선택자는 아니지만 엘리먼트들의 상태에 따라서 마치 클래스 선택자처럼 여러 엘리먼트를 선택할 수 있다는 점에서 붙은 이름입니다.

  • 링크와 관련된 가상 클래스 선택자

    여러 선택자가 중첩될 경우 마지막 선택자의 효과가 적용되므로 순서대로 지정하는 것이 좋습니다.

    • :link - 방문한 적이 없는 링크
    • :visited - 방문한 적이 있는 링크
    • :hover - 마우스를 롤오버 했을 때
    • :active - 마우스를 클릭했을 때

3.CSS 속성 (추가)

  • font-size : 폰트의 사이즈를 지정합니다.

    • rem : html 태그에 적용된 font-size의 영향을 받습니다. html 태그의 폰트 크기에 따라서 상대적으로 크기가 결정되기 때문에 이해하기 쉽습니다.

    • px : 모니터 상의 화소 하나의 크기에 대응되는 단위입니다. 고정된 값이기 때문에 이해하기 쉽습니다만, 사용자가 글꼴의 크기를 조정할 수 없기 때문에 가급적 사용을 하지 않는 것이 좋습니다.

조화

상속

모든 엘리먼트의 속성을 하나 하나 지정하는 것보다 상속을 이용하면 훨씬 효율적입니다. 아래처럼 html의 서체 색상만 조정하면 하위에 있는 모든 엘리먼트의 색상이 자동으로 바뀌게 됩니다. 이것은 각 엘리먼트가 상위 엘리먼트의 서체 색상 값을 물려 받기 때문입니다.

이러한 기능을 상속(inheritance)이라고 합니다.

모든 속성이 상속을 지원하는 것은 아닙니다. 상속을 하면 오히려 비효율적인 속성들이 있으므로 아래의 사이트에서 그 목록을 확인합니다.
https://www.w3.org/TR/CSS21/propidx.html

stylish

웹사이트의 디자인을 사용자 마음대로 수정할 수 있는 기능입니다. 이 기능을 이용해서 주어진데로 웹페이지를 소비하는 것이 아니라 자신의 취향을 반영할 수 있습니다. 또 다른 사람이 만든 디자인을 적용해서 간편하게 사이트의 디자인을 변경할 수도 있습니다.
https://userstyles.org/

캐스케이딩

하나의 엘리먼트에 대해서 다양한 효과가 영향력을 행사하려고 할 때 우선순위를 어떻게 설정하는가에 대한 규칙입니다.

웹브라우저 < 사용자 < 저자 순으로 영향력이 큽니다.

tag < class < id < style 순으로 영향력이 큽니다. 구체적일 수록 영향력이 커집니다.

하지만 !important를 선언한다면 그 효과를 먼저 적용합니다.

레이아웃

html 엘리먼트들은 크게 두가지로 구분됩니다.

  • 화면 전체를 사용하는 태그 => block element
  • 화면의 일부를 차지하는 태그 => inline level element

박스모델

밖에서부터 margin-border-padding-width,height(content 영역) 순으로 적용됩니다.

box-sizing : border-box 를 설정하면 border를 기준으로 width를 지정하여 직관적입니다.

마진겹침 현상(주의)

마진값 끼리 겹치는 경우에는 더 큰 쪽의 마진값이 두 엘리멘트의 사이값이 됩니다.

포지션

엘리먼트의 위치를 지정하는 방법입니다.

  • static : 디폴트 값으로, 원래 있어야 할 자리에 고정됩니다. top,left 등의 값에 영향을 받지 않습니다.
  • relative : 엘리멘트가 원래 있어야 할 자리를 기준으로 위치를 이동합니다.
  • absolute : 브라우저 창 좌측 상단을 기준으로 오프셋을 지정합니다. 기본 위치는 원래 있어야 할 자리입니다. 상위 태그에 static이 아닌 부모가 나타나면 그 위치를 기준으로 오프셋을 지정합니다. 부모와 링크를 끊기 때문에 자신의 컨텐츠 크기를 가지고, 부모도 자식을 없는 것처럼 취급합니다.
  • fixed : 화면에 고정시킵니다. width와 height를 지정하지 않으면 부모가 없어지기 때문에 자기 컨텐츠 크기를 표시합니다.
  • float : 이미지 위치 지정 또는 레이아웃을 잡는 데에 사용됩니다. 다른 요소가 해당 엘리멘트를 피해 위치하도록 합니다.
  • clear : float의 효과를 무시합니다.
