JavaScript

1.JavaScript란?

Javascript는 HTML과 CSS로 만들어진 웹페이지를 동적으로 변경해주는 언어입니다. Javascript는 기능이 별로 없는 언어이나 변수, 반복, 조건, 함수 심지어 객체까지 모두 가지고 있는 본격적인 프로그래밍 언어입니다.

JavaScript와 HTML이 만나는 법

  • Script 태그

  • 이벤트

  • 콘솔

    웹 브라우저의 콘솔 창에서 JS코드를 작성하여 적용할 수 있습니다.

2.기본적인 문법

데이터타입

  • 변수와 대입 연산자

  • 문자열
    따옴표(“ “) 로 묶어 작성합니다. 밑의 링크에서 사용 가능한 속성의 목록을 확인 가능합니다.
    https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String

  • 숫자
    산술연산자(+, -, *, /)를 사용할 수 있습니다.

  • Boolean
    True 와 False로, 참과 거짓을 나타냅니다.

    비교 연산자 : 두 값을 비교하여 Boolean 값을 반환합니다.

    • x === y : 두 값이 같은가?
    • x > y : x가 큰가?

  • 배열(array)

    수납상자 같은 데이터 구조입니다. [ , ] 형식으로 작성합니다. 인덱스(0부터 시작)으로 배열 안의 값을 가져옵니다. 문자열과 마찬가지로 사용할 수 있는 속성을 검색하여 찾아내는 것이 중요합니다. (keyword : javascript array ????)
    https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Array/length

조건문

c++의 문법과 유사합니다.

조건문을 사용하면 더 적은 엘리멘트를 사용하고도 원하는 효과를 나타낼 수 있습니다. 밑의 코드는 하나의 버튼으로 배경이 흑/백으로 전환되도록 하는 코드입니다.

위의 코드를 refactoring 해보겠습니다.
document.querySelector(‘body’) -> target 이라는 변수로 지정합니다.
document.querySelector(‘#night_day’)-> this 로 대체합니다.

반복문

마찬가지로 c++의 문법과 유사합니다. while을 사용합니다.

배열을 사용한 반복문입니다.
‘+ content +’ 는 속성값이나 컨텐츠 어느곳이나 사용 가능합니다.

또 다른 예시입니다. document.querySelectorAll(‘a’) 을 사용하여 전체 <a> 태그를 가져오고, 그 태그마다 반복문을 사용합니다.

함수

연속적이지 않은 반복에 대해서는 반복문을 사용하기가 어렵습니다. 이 경우 함수를 사용합니다.

인자는 함수에 들어가는 input입니다. 매개변수는 인자를 함수 내부로 매개해주는 역할을 합니다.

리턴은 함수 내부의 결과 값을 함수 밖으로 보내는 역할을 합니다. 함수 밖에서 사용이 가능합니다.

refactoring 한 코드입니다. 만약 onclick=”nightDayHandler()”로 넘겨주게 된다면 this가 전역변수로 지정되어 버튼에 적용되지 않게 됩니다. 따라서 this를 인자로 넘겨주고 self로 매개변수를 지정하는 테크닉을 사용합니다.

객체

서로 연관된 함수와 변수(메소드와 프로퍼티)들을 그룹화하는 컨테이너입니다. 비슷한 기능을 하는 함수들을 많이 사용하게 되면 함수들을 구별하기 위해 함수명이 길어지고, 알아보기 어려워집니다. 객체를 사용하면 이러한 문제를 간단히 해결할 수 있습니다.

완성된 코드입니다. 코드가 매우 길고 복잡할 경우에 이름이 중복될 수 있습니다. 함수를 수정할 때 미처 몰랐던 다른 곳까지 영향을 미칠 수 있습니다. 그래서 그 함수가 적용될 범위를 만들어 줍니다. 함수의 이름은 간결하게 유지해서 함수 이름만 보고도 기능을 명확히 알게 하고, 적용 대상을 객체로 분리합니다.

코드가 짧은 경우에는 큰 상관 없지만 코드가 복잡해지면 정리하는게 좋다고 합니다.

3. 기타

유지 및 보수

한 페이지에 html와 css, Javascript가 다 들어있다면 보기도 힘들고, 페이지의 수가 늘어날 경우 유지보수가 힘듭니다. 따라서 자바스크립트 파일을 따로 저장하고, src 속성을 이용하여 해당 파일을 html에 적용합니다.

라이브러리 vs 프레임워크?

라이브러리 : 재사용하기 좋은 부품입니다. 개발을 하다가 필요할 때에 적재적소에 사용 가능합니다.

프레임워크 : 구조와 방식이 정해져 있는 공장과 비슷합니다. 프레임워크의 동작 방식을 이해하고, 개발자는 프레임워크 내부의 엘리멘트들을 조합하여 효율적으로 개발을 할 수 있습니다.

두 개념을 구별짓는 기준은 ‘’도구의 영향력’’과 ‘’개발자와 도구 간 의존성’’으로 이해했습니다.

라이브러리는 여러 개발 프로젝트에서 특정 기능을 빠르게 만들기 위해 선택적으로 사용 가능합니다. 반면 프레임워크는 특정 목적을 달성하기 위해 만들어졌으므로, 프로젝트 전반에 걸쳐 영향력을 발휘합니다.

의존성 측면에서는 도구가 개발자에 의해 사용된다면 라이브러리고, 개발자가 도구를 통해 무언가를 만든다면 프레임워크입니다. 개발자가 라이브러리를 사용할 경우에 창의성이나 능력을 발휘할 수 있는 여지가 많아집니다.

UI vs API

UI(User interface) : 유저가 사용하고 보는 부분입니다. 일반 사용자가 앱의 동작과 기능을 제어하게 해줍니다.

API(Application programming interface) 개발자가 사용하고 보는 부분입니다. API를 통해 소프트웨어를 제어하고 개발합니다. (구글 지도, 결제, 로그인 등)
