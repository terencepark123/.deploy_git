PHP

1.PHP란?

  • 주로 HTML 코드를 프로그래밍적으로 생성합니다.
  • 서버쪽에서 실행 되는 프로그래밍 언어입니다.
  • Personal Home Page Tools 의 약자에서 PHP:Hypertext Preprocessor 로 의미가 변경 되었습니다.

PHP의 장점

  • 웹에 최적화된 언어입니다.
  • 웹개발에 필요한 수많은 로직들이 함수의 형태로 미리 제공됩니다.
  • 거의 모든 데이터베이스를 지원합니다.
  • 가장 많은 공개소프트웨어가 PHP로 만들어졌습니다.

PHP 동작원리

밑의 그림은 클라이언트와 서버 간에 어떠한 방식으로 PHP 파일을 처리하고, 파일을 전달하는 지 나타냅니다. HTML 파일을 처리하는 방식과 다르게, PHP파일은 PHP engine이 관여하여 HTML 문서를 생성합니다.

서버 측 언어를 사용하면 서버에서 데이터베이스를 처리할 수 있고, 동적으로 웹 페이지를 만들 수 있다는 장점이 있습니다.

CGI : 웹 서버와 PHP engine이 통신하는 규약입니다.

3.PHP 문법

데이터 타입

  • 숫자

    사칙연산이 가능합니다. var_dump 함수를 통해 data type를 확인 가능합니다. int, float 등이 있습니다.

  • 문자열

    문자열은 “ “를 통해 표시합니다. 또한 문자열의 합은 온점( . )을 사용하여 합쳐줍니다. 문자열 내에 따옴표를 사용하고자 할 경우에는 1. 해당 부분의 따옴표를 전체 문자열의 따옴표와 다르게 표기하거나(큰따옴표 - 작은 따옴표), 2. 백슬래쉬( \ ) 를 따옴표 앞에 삽입합니다.

    줄바꿈 할 때는 쌍따옴표 안에서 \n, \r을 사용합니다.

    문자열 안에서 변수를 사용하려면 쌍따옴표 안에서 { , }를 사용합니다. 또는 ‘ . ‘를 사용할 수 있습니다.

    정규 표현식은 문자열을 처리하는 방법 중의 하나로 특정한 조건의 문자를 검색하거나 치환하는 과정을 간편하게 처리하는 방법입니다.

    preg_match는 해당 문자열이 내용 안에 있는 지 여부를 boolean으로 반환합니다. i는 패턴 매칭을 할 때 대소문자를 구분하지 않도록 합니다. \b 는 단어의 경계를 의미합니다.

    ( ) 는 raw 문자를 의미합니다.

    \w 문자열을 의미합니다.

    +는 여러 문자를 의미합니다.

    @는 구분자입니다.
    ^은 행의 경계를 의미합니다. [ ]안에 담길 경우 부정을 의미합니다.
    ?:은 matches라는 변수에 담기지 않습니다.
    ?은 하나 미만을 의미합니다.

    ?p 는 서브패턴을 해당 키를 가진 값으로 저장합니다.

    preg_replace를 사용하면 string에 담긴 내용을 pattern으로 인식한 후, 해당 pattern중 특정 성분을 치환할 수 있습니다.

  • Boolean

    불린으로 올 수 있는 값은 true와 false 두가지 밖에 없습니다. 불린은 조건문에서 핵심적인 역할을 담당합니다. 조건문에서 관습적인 이유로 0는 false, 0이 아닌 숫자는 true를 대체할 수 있습니다.

  • 변수

    $로 표시합니다.

  • 배열

    배열은 여러 개의 데이터를 하나의 변수에 담아서 관리합니다. 변수를 선언할 때에는 [ ] 안에 데이터를 넣는 형식을 따릅니다. php 5.4 이전 버전에서는 array( ) 형식으로 선언하기 때문에 하위 호환성을 위해서는 아래의 형식을 사용하는 것이 권장됩니다.

    연관배열

    일반적으로 다른 언어에서는 숫자를 인덱스로 사용하는 것을 일반적으로 배열, 혹은 indexed array라고 하고, 문자를 인덱스로 사용하는 것을 연관배열(associative array, hash, dictionary)라고 부르지만 PHP에서는 이를 특별히 구분하지 않아 키 값으로 숫자나 문자를 모두 사용할 수 있습니다.

    배열에 아이템을 추가하는 함수입니다.

    배열의 아이템을 제거하는 함수입니다.

    배열의 아이템을 정렬하는 함수입니다.

  • 비교

    ==(같다), !=(같지 않다) 를 boolean 값으로 나타냅니다.

  • 입출력

    PHP를 사용하면 입력에 따라 출력이 다르게 동작할 수 있습니다. 밑의 코드는 URL로부터 받은 id와 password 값을 각각 출력합니다.

    해당 url을 받아온다면 다음과 같은 결과를 출력합니다.

    URL에 직접 정보를 입력하는 것은 불편하므로 HTML의 폼기능을 이용합니다. 폼의 내용을 입력하고 전송을 하면, URL을 통해 정보를 2.php로 보냅니다.

    • GET VS POST

      GET 방식으로 데이터를 전송할 때 URL에 데이터를 포함시키는 것이 비해서 POST 방식은 HTTP 메세지에 정보를 숨겨서 전송합니다. 이러한 차이로 인해서 GET 방식은 정보에 대한 링크로 많이 사용되고, POST 방식은 사용자의 아이디나 비밀번호와 같은 데이터를 전송하는 방식으로 주로 사용합니다.

  • 조건문

    주어진 조건에 따라서 에플리케이션을 다르게 동작하도록 합니다. 다음과 같은 형식을 따릅니다. else if를 통해 다양한 케이스를 복수로 설정합니다.

    다음은 예제 PHP 애플리케이션입니다. 10.html로 접속하여 id를 입력하고 제출을 클릭하면, 11.php를 php engine이 해석하여 html을 만들고, 다시 클라이언트에게 제공합니다.

  • 반복문

    while 문은 조건이 참(true)이면 중괄호 구간의 구문을 반복적으로 실행합니다. 조건이 false면 반복문이 실행되지 않습니다. 여기서 true와 false는 종료조건이 되는데, 반복문에서 종료조건을 잘못 지정하면 무한반복이 되거나, 반복문이 실행되지 않습니다.

    주로 두가지 테크닉 중 하나를 사용하는데, 먼저 변수 i를 whle loop 밖에 선언하고, while 문 내부에서 i값을 하나씩 증가시켜 해당 값이 특정 값을 넘을 경우 종료하는 방식이 있습니다.

    다른 하나로는 for문을 이용하는데, 특정 변수를 초기화한 후 i 값을 특정 값을 넘지 않을 때 까지 증가시키며 중괄호 내의 내용을 반복합니다.

    반복문을 중단시키고 싶을 경우에는 내부에 break을 사용합니다. while 문에서 loop를 빠져나오는 수단으로 사용하기도 합니다.

    continue는 반복문이 돌아가는 중에 실행을 중단시키고, i를 증가시켜 다음 실행을 진행합니다. 밑의 코드에서는 i 값이 5일 경우에 실행이 되지 않으나, 그 이후부터는 정상적으로 실행이 된 결과를 출력합니다.

    for 문이나 while 문을 여려 개를 사용하여 중첩할 수 있습니다. 복잡도는 증가합니다.

  • 함수

    함수의 핵심은 입력과 출력입니다. 입력된 값을 연산해서 출력하는 것이 함수의 기본적인 역할입니다.

    함수는 키워드 function 뒤에 함수의 이름이 오고, 괄호가 따라옵니다. 괄호 안에는 함수에 대한 입력 값을 담을 변수의 이름이 오고, 복수의 입력 값을 정의 할 때는 변수와 변수 사이에 콤마로 구분합니다. 중괄호에는 함수가 호출 되었을 때 실행될 코드가 옵니다.

    함수가 없었다면 특정 기능을 반복해야 하는 경우 같은 코드를 여러 번 반복해야 하기 때문에 문서가 길어집니다. 함수를 사용하면 한 번의 호출로 동일한 기능을 수행하여 효율적입니다.

  • include

    재활용성을 높이고, 유지보수를 쉽게 할 수 있는 방법 중 하나가 코드를 여러개의 파일로 분리하는 것입니다. 이를 통해서 얻을 수 있는 효과는 아래와 같습니다.

    • 자주 사용되는 코드를 별도의 파일로 만들어서 필요할 때마다 재활용할 수 있습니다.
    • 코드를 개선하면 이를 사용하고 있는 모든 애플리케이션의 동작이 개선됩니다.
    • 코드 수정 시에 필요한 로직을 빠르게 찾을 수 있습니다.
    • 필요한 로직만을 로드해서 메모리의 낭비를 줄일 수 있습니다.

    다음과 같이 두개의 파일로 나눠 작성한 후, include를 통해 함수 welcome을 불러옵니다.

    PHP는 외부의 php 파일을 로드하는 방법으로 4가지 형식을 제공합니다. 존재하지 않는 파일의 로드를 시도했을 때 include가 warning를 일으킨다면 require는 fatal error를 일으킵니다 fatal error는 warning 보다 심각한 에러이기 때문에 require가 include 보다 엄격한 로드 방법입니다.

    _once라는 접미사가 붙은 것은 파일을 로드 할 때 단 한번만 로드하면 된다는 의미입니다. 자원 낭비를 막을 수 있습니다.

    • include
    • include_once
    • require
    • require_once

  • namespace

    PHP에서는 함수의 중복 선언을 허용하지 않습니다. 같은 파일에 동일한 이름을 가진 두개의 함수를 import 했을 경우 사용합니다. 로드되는 파일의 초입에 키워드 namespace를 이용해서 네임스페이스를 만듭니다. 그리고 네임스페이스를 사용할 때는 함수 앞에 네임스페이스의 이름을 붙여서 사용합니다.

  • 라이브러리와 확장기능

    라이브러리에서 제공하는 API를 이용해서 PHP엔진이 제공하는 기능을 자신이 만들고 있는 응용 프로그램에서 사용할 수 있습니다. phpinfo, date와 같은 함수를 언어에 기본적으로 내장되어 있다고 내장함수라고도 부릅니다.

    PHP는 웹개발에 필요할만한 수 많은 내장함수를 제공합니다. 이 내장함수들은 빠르게 웹에플리케이션을 구축하는데 도움을 주고, 직접 구현하는 것보다 실행속도가 빠르기 때문에 어떤 기능을 구현하기에 앞서서 API가 있는지 찾아보아야 합니다.

파일 다루기

  • 파일의 제어

    copy는 복사가 성공하였을 경우 True를 반환합니다. 이를 이용하여 실패하였을 경우 메세지를 띄우는 예제 코드입니다.

    파일을 삭제하는 코드입니다.

    파일을 읽고 문자열을 리턴하는 코드입니다. 웹 상의 데이터를 읽어올 수도 있습니다.

    파일을 읽고 문자열을 저장하는 코드입니다.

    파일을 읽고 쓸 때 권한의 문제로 오류가 발생할 수 있습니다.

    특정 파일이 읽을 수 있는 상태인 지 확인하는 코드입니다.

    특정 파일이 쓰기가 가능한 지 확인하는 코드입니다.

    파일이 있는 지를 확인할 수 있습니다.

  • 디렉토리

    현재 작업이 이루어지는 디렉토리를 알려주는 getcwd 함수입니다. chdir은 디렉토리를 변경합니다. scandir은 현재 디렉토리의 파일 이름을 불러옵니다.

    mkdir은 디렉토리를 생성합니다. 첫번째 인자로 디렉토리의 이름, 두번째 인자로 디렉토리의 권한을 지정, 세번째 인자로 디렉토리를 한번에 생성하는 기능을 제공합니다.(재귀적)

세션

디버그

  • log

    • tail : 파일의 끝부분부터 거꾸로 읽어서 출력하는 명령어,
      -f 옵션을 사용하면 실시간으로 파일의 변경사항을 열람할 수 있다.
    • access log : (html, css, js, image와 같은) 리소스가 웹서버를 통해서 로드 됐을 때 기록되는 정보
      • bitnami : /opt/bitnami/apache2/logs/access_log
    • error log : 오류가 발생했을 때 기록되는 정보
      • bitnami : /opt/bitnami/apache2/logs/error_log
    • tail -f /opt/bitnami/apache2/logs/access_log
      -f 는 로그가 바뀔 때마다 알려줍니다.
    • error_log : 로그로 메시지를 보낸다.
    • error_log(var_export(값, 1)); : 로그로 메시지를 보내는데 값이 객체라면 읽을 수 있는 형식으로 전환한다.

    var_dump를 사용하여 흐름 제어를 하게 되면 어떤 로직이 실행되고 있는 지 확인 가능합니다.
